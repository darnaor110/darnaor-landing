# Core Frameworks

## עקרון-על: Hook → Script → Format → Editing (בסדר הזה תמיד)

**הסדר חשוב ביותר.** רוב האנשים מתחילים מהפורמט ("אני אעשה ראש מרחף") — שגוי. מתחילים מההוק, ואז כותבים תסריט, ואז בוחרים פורמט.

## מחקר אאוטליירים — הבסיס לכל

**The 5x Rule:** תוכן שמצליח הוא כזה שמקבל לפחות 5 פעמים יותר צפיות ממספר העוקבים. זה הסימן שהוא "פרץ" את הבועה.

**שיטת המחקר:**
- 20 דקות גלילה מכוונת ביום — לא לצריכה, לחקר
- מיפוי: Hook | צפיות | נושא | פורמט
- לחפש דפוסים: מה משותף לתוכן שפורץ בנישה שלך?
- **לא לחקות תוכן — לחקות פורמטים.** תוכן = שלך. מבנה + פורמט = מותר לגנוב.

## Three-Layer Hook Architecture

**שלוש שכבות בוך 3 שניות ראשונות:**
1. **משהו לראות** (Visual) — טקסט צף, גרפיקה, תנועה, אקסשן פיזי
2. **משהו לקרוא** (Text on screen) — כותרת, שאלה, טענה
3. **משהו לשמוע** (Audio) — המשפט הראשון שנאמר

**הוויזואל נושא יותר משקל מהמילים הנאמרות.** אם הטקסט על המסך חזק אבל מה שנאמר רגיל — עדיין עובד. להיפך — פחות.

## עיקרון Broad → Narrow → Niche

```
HOOK (רחב) → ערך (ממוקד לאווטר שלך) → CTA (נישתי מאוד)
```

**דוגמה:**
- Hook: "למה כולם נכשלים בחיטוב" (רחב — מושך כולם)
- ערך: "המתאמן שכבר שנה לא רואה תוצאות עושה את הטעות הזאת..." (ממוקד לאווטר)
- CTA: "אם אתם מתאמנים כבר שנה ולא רואים תוצאות — שלחו לי 'חיטוב'" (נישתי)

**כלל ברזל:** אל תנישה את ה-Hook. Hook שנשמע כמו פרסומת ("מאמן כושר? תראה את זה") = מת.

## מבנה תסריט לפי PBL (5 רכיבים)

1. **Notes** — פורמט + הנחיות צילום כלליות
2. **Bulleted Script** — שורה שורה, מה אומרים בדיוק
3. **Caption** — טקסט לפוסט
4. **Subtitle specs** — איך הכתוביות נראות
5. **Editing notes** — הנחיות לעורך: גרפיקות, קאטים, בי-רול

## פורמטים שעובדים (Format Skins)

- **Talking Head** — בונה אמון, האישי ביותר
- **Whiteboard / גרפיקות** — מועיל לתוכן שצריך ויזואליזציה
- **ABR (Audio B-Roll)** — קריינות על שוטים, מהיר לצילום
- **Two-person dialogue** — מישהו חכם מלמד מישהו טיפש, שאלה/תשובה
- **Screen recording** — אפליקציות, מחקרים, נתונים

**Clone Strategy:** כשתסריט מצליח — צלם אותו שוב בכל הפורמטים האחרים. החלף שמות/מספרים (nouns) — שמור את הליבה.

## פאנל ההמרה של PBL

```
Views (צפיות) 
  → Watch Time (אחוז שמירה) 
    → Followers (עוקבים) 
      → Comment Lead (הגיבו X) 
        → ManyChat DM 
          → דף נחיתה עם הוכחות 
            → VSL (סרטון מכירה) 
              → שיחת היכרות → לקוח
```

**Comments > Follows > Likes** — תגובות הן המטבע הכי יקר, הכי עמוק להכנסת אדם לפאנל.

## Stories = קופה יומית

Stories לא לידעת — לחימום + לידים יומיומיים:
- לפני/אחרי (תמונה, לא טקסט)
- שאלה פשוטה שמזמינה תגובה
- קישור לסרטון שעלה
- "אם אתם רוצים X — ענו לי כאן"

## חוק הקריאות

כתוב כאילו הקהל שלך בן 12. משפטים קצרים. מילים פשוטות. **"המון אנשים מבינים פחות ממה שחושבים."** אם אתה צריך לחשוב על המשמעות — יש לפשט.

## טעויות נפוצות

- לחקות תוכן ולא פורמט
- Hook שנשמע כמו פרסומת / נישתי מדי
- לקנות עוקבים (מחריב אלגוריתם)
- לתת לAI לכתוב תסריטים סופיים — חייב לעבור דרכך
- לא למדוד: אם לא מסתכלים על מה שעובד — עובדים בעיוורון

## האווטר (הקהל הספציפי)

אנשים אנליטיים (20-35) שכבר מתאמנים במכון מעל שנה, משקיעים בתזונה, **אבל תקועים** — לא רואים תוצאות ולא מבינים למה. אינטלקטואלים שרוצים להבין, לא רק לבצע.

**כאב עמוק:** "אני משקיע הרבה ולא מתקדם — אולי אני פשוט לא מסוגל?"
**חלום:** גוף חטוב, מושך, בריא — ולשמור עליו לבד, לנצח.

## קול המותג

- חבר חכם שמלמד, לא מרצה
- ספציפי: מספרים, זמנים, תוצאות קונקרטיות
- מבוסס מדע אבל מנגיש: "המחקר אומר X, ובפועל זה אומר Y"
- אותנטי: מספר על הכישלונות שלו, לא רק הצלחות
- לא מוכר פנטזיה — מוכר ריאליות עם תוצאות

## עמודי תוכן עיקריים

1. **תזונה וחיטוב** — מיתוסים, קלוריות, פרוטוקולים, מתכונים

4. **הסיפור האישי שלי** — מסע, קשיים, לפני/אחרי, גילויים
5. **"חיטוב עם דר"** — תיעוד זמן אמת של חיטוב (סדרה שוטפת)

# Common Tasks

## כתיבת Hook Stack

כשמבקשים hook — קרא `references/hooks-library.md` ואז:

1. **זהה את הקטגוריה הנכונה** לפי הנושא
2. **הצע 3 אפשרויות** מותאמות — אחת עם כוונה שונה כל אחת
3. **לכל אחת ספק Hook Stack מלא:**
   - **Verbal hook** — מה שומעים ב-3 השניות הראשונות
   - **Written hook** — הטקסט על המסך ב-3 השניות הראשונות
   - **Visual hook** — מה רואים ב-3 השניות הראשונות
4. הסבר לאיזו זווית כל hook שייך ולמה

## כתיבת Script

כשמבקשים סקריפט, ציין:
- איזה מ-3 סוגי התוכן זה (Educational / Storytelling / Authority)
- איזה script angle נבחר ומדוע
- מהו ה-format המומלץ לצילום
- מה ה-CTA ולמה

## אסטרטגיית תוכן

כשמבקשים אסטרטגיה, ציין:
- היכן Dar Naor נמצא (0–1K / 1K–10K / 10K+)
- את תמהיל ה-TOFU/MOFU/BOFU המתאים
- את הכלל (70/20/10 או 90/10) לפי השלב

## Audit תוכן

כשמבקשים לסקור תוכן קיים, עבור לפי:
1. Hook — האם יש Hook Stack מלא? האם הוא outlier-based?
2. Value — measurable או fluffy?
3. Format — מעניין או "sitting and talking"?
4. CTA — ברור? נכון לשלב?
